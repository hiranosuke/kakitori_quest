# kakitori_quest 開発ガイドライン

## コーディング規約

### ハードコード禁止

固有名詞・数値パラメーター・文字列定数は、必ず設定ファイルや定数モジュールに切り出す。
インラインのマジックナンバーや文字列リテラルは使わない。

**対象の例：**
- ゲームパラメーター（ゴールド量、ハート数、アイテム価格など）
- メッセージ文字列
- アイテム・種族・スロットの名称
- SVG座標・サイズ定数

**定義場所の原則：**
| 種別 | 定義場所 |
|---|---|
| ゲームバランス数値 | `src/config/rewards.ts` など `config/` 配下 |
| 表示文字列・メッセージ | `src/store/messages.ts` |
| アイテム定義 | `src/config/items.ts` |
| スロット定義 | `src/config/decorationSlots.ts` |
| SVG描画定数 | 各コンポーネント内の定数オブジェクト（ファイル先頭） |

**やむを得ずハードコードが必要な場合：**
実装前にユーザーに理由を説明し、明示的な許可を得ること。

### アイテム説明への効果記載禁止

`src/config/items.ts` の `description` フィールドには、**ゲームパラメーターとして未実装の効果（まもり上昇・まりょくアップなど）を記載しない**。
現在実装されているのは「かいふく」のみ。新たな効果パラメーターを実装する前に description に記載すると誤解を招くため禁止。
